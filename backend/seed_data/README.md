# Seed Data — Ecommerce Showcase MVP

This folder contains pre-generated CSV seed data for the BigQuery `ecommerce_showcase` dataset.

## Files

| File | Rows | Description |
|------|------|-------------|
| `sales_orders.csv` | ~5,300 | 90 days of order data across 20 products, 6 countries, 3 channels |
| `products.csv` | 20 | Product catalog with prices, costs, and stock levels |
| `ad_performance.csv` | ~546 | Daily performance for 6 campaigns across Google, Meta, TikTok |
| `warnings.csv` | 5 | Pre-generated showcase alerts |
| `business_health.csv` | 8 | Business domain health status indicators |
| `conversations.csv` | 5 | Past Ask assistant conversations with realistic Q&A |
| `deep_review_scenarios.csv` | 4 | Hardcoded multi-model review scenarios |
| `settings.csv` | 12 | UI settings and threshold values |

## Table Relationships

```
products.product_id ←── sales_orders.product_id
warnings.deep_review_scenario_id ──→ deep_review_scenarios.scenario_id
warnings.category ←──→ business_health.domain (matching domains)
```

## Date Range

All data uses **2026 dates** (current year):
- Full range: March 11, 2026 → June 9, 2026 (90 days)
- Supports Last 7 / Last 30 / Last 90 day views

## Demo Story

The seed data tells a coherent ecommerce story with 5 active issues:

### 1. Low Stock: Running Shoes Pro (WARN-001 → SCENARIO-001)
- `products.csv`: PROD-009 has `stock_quantity: 38`
- `sales_orders.csv`: ~8 orders/day for this product
- `business_health.csv`: `inventory` domain is `critical`
- `deep_review_scenarios.csv`: All 3 models agree — expedited reorder needed

### 2. Ad Spend Inefficiency: Brand Awareness EU (WARN-002 → SCENARIO-002)
- `ad_performance.csv`: CAMP-02 has ROAS ~1.1x in last 14 days (spend +30%, revenue -30%)
- `ad_performance.csv`: Other campaigns (CAMP-03, CAMP-05) perform well by contrast
- `business_health.csv`: `advertising` domain is `warning`
- `deep_review_scenarios.csv`: Models partially agree — one suggests reduced spend vs full pause

### 3. Refund Spike: Phone Case Classic (WARN-003 → SCENARIO-003)
- `sales_orders.csv`: PROD-005 refund rate is ~12% in last 14 days (vs 4% baseline)
- `business_health.csv`: `returns` domain is `warning`
- `deep_review_scenarios.csv`: Models agree — it's a listing accuracy issue

### 4. Payment Decline Rate (WARN-004 → SCENARIO-004)
- `business_health.csv`: `payments` domain is `warning`
- Narrative: decline rate up from 1.5% to 3.2%, mainly AT/CH credit cards
- `deep_review_scenarios.csv`: Models partially agree on urgency

### 5. Listing Suppressed: Travel Backpack (WARN-005, no Deep Review)
- `products.csv`: PROD-020 has `stock_quantity: 18`
- `business_health.csv`: `listings` domain is `warning`
- No Deep Review scenario linked (lower priority issue)

## Functional vs Mocked

| File | Status | Notes |
|------|--------|-------|
| `sales_orders.csv` | **Functional** | Real data for BigQuery queries, KPIs, charts, Ask |
| `products.csv` | **Functional** | Real data for product analytics and stock alerts |
| `ad_performance.csv` | **Functional** | Real data for ROAS, ACoS, ad spend analytics |
| `warnings.csv` | **Mocked** | Pre-seeded, not generated by detection logic |
| `business_health.csv` | **Mocked** | Static status tiles, manually aligned with warnings |
| `conversations.csv` | **Functional** | Realistic past conversations for Ask sidebar |
| `deep_review_scenarios.csv` | **Mocked** | Hardcoded multi-model consensus, no real AI |
| `settings.csv` | **Partially Functional** | UI persistence values, no real backend control |

## Regenerating Data

To regenerate all CSVs from scratch:

```bash
python3 backend/seed_data/generate_seed_data.py
```

The script uses `random.seed(42)` for reproducible output.
